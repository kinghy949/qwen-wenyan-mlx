# 后续可扩展方向

基于当前 Run #3 adapter（白话↔文言互译），可以从四个方向延伸。按"投入产出比 + 好玩程度"排序。

---

## 🚀 A. 即时能玩 / 给人用

### A1. Gradio Web Demo
- ~30 行代码：`gradio` 起一个输入框 + 输出框，调用 `mlx_lm.generate`
- `share=True` 自动生成公网链接，发朋友圈 / 群里玩
- **预估投入**：半天

### A2. Mac 菜单栏 / Raycast / Alfred 小工具
- 选中任何文字 → 快捷键 → 文言化 → 粘贴回去
- 推荐架构：常驻 `mlx_lm.server`（OpenAI 兼容接口）+ Raycast 扩展（TypeScript）
- 备选：Apple Shortcuts（系统自带、零安装）
- **预估投入**：1 天

### A3. Telegram / 飞书机器人
- 群里 `@bot 这句话` → 返回文言版
- 同样跑常驻 `mlx_lm.server`，bot 框架做转发
- **预估投入**：半天

### A4. 浏览器插件
- 选中网页文字 → 右键 → "文言化" → 弹窗显示
- Chrome MV3 extension + 本地 HTTP 调用
- **预估投入**：1-2 天

---

## 🧠 B. 训新能力

### B1. 作者风格化（最有传播力）⭐
- 数据：用 GPT-4/Claude 批量造 200 条"白话 → 鲁迅风 / 王小波风 / 金庸风"对照
- 训出能模仿三种风格的独立 adapter，按需切换
- 应用："今天天气真好" → 鲁迅风：但我知道，那只是一种自欺
- **预估投入**：1-2 天（含数据生成）

### B2. 历史人物对话扮演
- 数据：孔子 / 苏轼 / 李白的语录 + 现代问题 → 用其语气作答
- 应用："和孔子聊聊我该不该跳槽"
- **预估投入**：2-3 天

### B3. 古诗续写 / 命题作诗
- 数据：《全唐诗》《全宋词》开源语料，约 3000 条够用
- 训出"给开头 / 给题目 → 生成完整古诗"
- **预估投入**：1-2 天

### B4. 古文阅读理解 QA
- 输入古文段落 + 问题 → 返回答案
- 古代版"机器阅读理解"
- **预估投入**：2 天

---

## 🔬 C. 工程化 / 更专业的训练流水线

### C1. LLM-as-Judge 自动评估 ⭐
- 让 Claude/GPT-4 对每条输出从"古意 / 通顺 / 忠实"三维度打 1-5 分
- 取代手工看 14 条的方式，能精准对比 20 个 checkpoint
- 解决 Run #1/#2/#3 主观对比难量化的痛点
- **预估投入**：半天到 1 天

### C2. 数据扩充流水线
- 大模型批量生成 1000 条 → 去重 → 长度筛 → C1 LLM-Judge 筛优 → train v2
- 是 Run #4 真正突破现有盲区（百分数 / 编程符号 / 网络梗反讽）的关键
- **预估投入**：1-2 天

### C3. 量化 + 上传 HuggingFace
- `mlx_lm.convert -q` 出 4bit 版本（约 400MB）
- 上传 HF / ModelScope，README 加 "直接 pip install 调用" badge
- **预估投入**：半天

### C4. DPO 偏好对齐
- 人工标几十组"哪个翻译更好"的偏好对 → DPO 再训一轮
- 让模型学风格偏好（简洁派 vs 文采派）
- **预估投入**：2-3 天

## 🎨 D. 跨界 / 多模态

### D1. 古籍 OCR + 翻译
- 拍照 → PaddleOCR / Apple Vision → 当前模型 → 现代白话
- 做成 iOS App 或 Mac App
- **预估投入**：3-5 天

### D2. 文言朗读

- 应用："AI 朗诵《滕王阁序》"
- **预估投入**：1 天

### D3. 古诗配画

- **预估投入**：2 天
