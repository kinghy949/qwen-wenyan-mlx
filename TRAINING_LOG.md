# 训练日志

## Run #1 — 2026-05-19

### 配置

- 基座：`Qwen/Qwen2.5-1.5B-Instruct`（本地，ModelScope 下载）
- 数据：142 train / 10 valid / 5 test（chat messages 格式）
- LoRA：rank=8, scale=20, dropout=0, num_layers=8
- 训练：batch_size=2, iters=600, lr=1e-5, max_seq_length=1024
- 硬件：MacBook Air M5（16GB / 10-core GPU）
- 可训练参数：**2.638M / 1543.714M ≈ 0.171%**
- 训练时长：约 1.5 分钟（峰值显存 3.95GB）

### Loss 曲线

| Iter | Train loss | Val loss | 备注 |
|---:|---:|---:|---|
| 1   | —     | **3.148** | 起点 |
| 100 | 0.493 | **0.629** | ⭐ Val 最低点 |
| 200 | 0.343 | 0.679 | |
| 300 | 0.140 | 0.733 | |
| 400 | 0.114 | 0.784 | |
| 500 | 0.096 | 0.804 | |
| 600 | 0.084 | **0.840** | 最终（已过拟合）|

### 诊断：典型过拟合

- Train loss 从 1.418 → 0.084，跌了 **17×**。
- Val loss 从 iter 100 的 0.629 单调上升到 0.840。
- 训练 loss 还在降、验证 loss 反而升 → 模型在**背训练样本**，泛化能力下降。
- 根因：142 条样本 × batch 2 = 71 步/epoch；600 iters ≈ **8.5 个 epoch**，对小数据集太多。

### 应对：切回 val 最低的 checkpoint

mlx-lm 每 200 步保存一次，目录里有：
- `adapters/0000200_adapters.safetensors` ← 离最佳点最近（val 0.679）
- `adapters/0000400_adapters.safetensors`
- `adapters/0000600_adapters.safetensors` ← 当前 `adapters.safetensors`

切换最佳 checkpoint：
```bash
cp adapters/0000200_adapters.safetensors adapters/adapters.safetensors
bash scripts/generate.sh "请将以下白话文翻译为文言文：今天加班到很晚。"
```

### 其他观察

- **显存极度富裕**：3.95GB / 16GB。完全可以上 batch_size=4 或换 3B 基座。
- **训练超快**：600 步约 90 秒（~7 it/s）。下一轮可以多跑几组超参对比。

### 测试集评估（iter 600 final adapter）

```
Test loss 0.858, Test ppl 2.359
```

- Test loss (0.858) ≈ Val loss (0.840)，两者一致 → 测试集和验证集分布相近，**数字真实反映过拟合后的泛化水平**。
- Perplexity 2.36 含义：模型对正确答案的平均"分支预测数"约 2.4 个 token，对生成任务属于**可用但不惊艳**的水平（理想 < 2.0）。
- 用 iter 200 的 adapter 重测，理论上 test loss 应该降到 ~0.68 附近——见下方对照。

### Checkpoint 对照（同一测试集 + 同一 prompt）

| 指标 | iter 200 (val 最低) | iter 600 (final) |
|---|---|---|
| Test loss | **0.640** ✅ | 0.858 |
| Test ppl  | **1.896** ✅ | 2.359 |
| Peak mem  | 3.205 GB | 3.218 GB |
| 输出 tokens | 18 | 23 |

Prompt：`请将以下白话文翻译为文言文：今天加班到很晚，地铁都停运了，只能打车回家。`

- **iter 200 输出**：`今日夜深方归，其时地铁已歇，唯有雇车而已。`
  - 简洁、地道——"其时"、"而已" 是真正的文言虚词。
- **iter 600 输出**：`今日值守至夜深，地铁已尽停载，孤身一人，唯有雇车归家。`
  - 多出的"孤身一人"是训练数据里没要求的修饰，**过拟合后模型在套训练集里的修辞模板**。

困惑度 2.36 → 1.90 是约 20% 的提升，且 1.90 已踏进"流畅"门槛（< 2.0）。**结论：早停（iter 200）的 checkpoint 才是 Run #1 真正的产物**。

### 主观质量对比（同 14 条 prompt）

诚实记录：**指标改善 ≠ 各方面都变好**。Run #2 在某些方向退步了。

#### ✅ Run #2 更好

| Prompt | Run #1 | Run #2 |
|---|---|---|
| 礼记 大道之行 | 大治的时候，天下是公共的…… | **大同社会**，天下为公，选拔贤人与能人…… |
| 早睡早起 | 早睡早起。（未改）| **早寝早起。**（一字之变，更文言）|
| 长新闻 | 据报，今年五月一日小长假之日，全国铁路客流量创历年新高，达两千万人次之多。 | 据报，五月一日小长假期间，全国铁路客流量创历史新高，达两千万人次之多。（更顺）|
| 朋友千里 | 友人千里**来访** | 友人千里**相访**（更地道）|

#### ❌ Run #2 反而退步

| Prompt | Run #1 ⭐ | Run #2 ❌ |
|---|---|---|
| 短视频 | 吾一夜而观诸**短影音** | 吾一夜刷**抖音**，至晓方**方睡**（造词丢了 + 错字）|
| 程序员分号 | 至三更方悟，少写一**逗**之故 | 至三更方悟，**少写了一个分号**（直接中文未翻译）|
| 反讽画饼 | **君画饼甚大甚圆**，可惜不能食 | 君画饼**又大又圆**，可惜不能食（白话残留）|
| 网络梗起飞 | …**当速飞升** | …**便只能原地踏步**（含义反了）|
| AI 改变生活 | 潜移默化之中，已渐染于吾辈之工作与生活之诸方 | 人之智巧，日日进步，其功业日日可观（脱离原意）|
| 哲思 | 非任其欲为，惟慎其独而不为 | 非心所欲，安能为也，唯心所欲，安能止也（语法不通）|

#### 🟡 仍然没补上的盲区

- "百分之四十" 还是翻成 "四十之百分比"——虽然训练数据里写了"增三成""二十之一"等，但模型没建立"百分之 X"这种字面输入到雅化输出的映射。
- 古诗 / 兵法反向翻译（举头望明月、知己知彼）仍然原样输出。补的 10 条诗词反译没起作用，可能 prompt 模式被模型当成"已是白话"。

### 解读：为什么指标涨但部分能力降？

1. **容量减半 (rank 8→4)** 让模型对**惊艳搭配**（"短影音""潜移默化"）的拟合能力下降——这些是 Run #1 偏过拟合时才学到的"高分作答"。
2. **Dropout** 引入随机性，让原来稳定的输出（君画饼**甚**大**甚**圆）变得不稳定。
3. **补的 80 条样本中混入了"几乎不变"的对照**（如"知己知彼，百战不殆"→几乎照搬，"早睡早起"→"夙兴夜寐"但格式相近），强化了"输入≈输出"模式，反而让模型在古诗反译时更倾向不动。

### 结论

- **如果优化目标是 perplexity**，Run #2 胜出，可以作为基线发布。
- **如果优化目标是"惊艳例子"**，Run #1 的 iter 200 反而是更好的产物——可控的过拟合在生成任务中有时是优点。
- **下一步可尝试**：折中超参（rank=6, dropout=0.02, iters=300），或者重新审视补充数据集，把"几乎照搬"的样本剔除。

#### ⭐ Run #3 突破点（Run #2 没补上的盲区被打开）

| Prompt | Run #2 ❌ | Run #3 ✅ |
|---|---|---|
| 古诗反译（举头望明月） | 举头望明月，低头思故乡。（照搬）| **抬起头来望着那轮明月，低下头来思念起故乡。** |
| 兵法反译（知己知彼） | 知己知彼，百战不殆。（照搬）| **了解自己，也了解对方，上百次的战斗都不会失败。** |
| 极简（早睡早起） | 早寝早起。 | **夙兴夜寐。**（教科书级雅化）|
| AI 概念 | 人之智巧，日日进步…… | **人工智巧，潜移默化，于吾辈之工作与生活，无处不改。**（集 #1 和 #2 之长）|
| 反讽画饼 | 君画饼**又大又圆**（白话残留）| **君画饼甚大且圆**，然不可食。（"且"比 Run#1 "甚...甚" 更顺）|
| 长新闻 | …达两千万人次之多 | …**逾两千万人次**（"逾"字精炼）|

#### 🟡 仍未解决

- **百分数**（01）：三轮都输出"四十之百分比"。即便训练集里写了"增三成""二十之一"，模型没建立"百分之 X"→雅化输出的映射。**根因可能是 prompt 模板差异**——训练样本里"百分之三十"对应"三成"，但 prompt 是直接含"百分之四十"，模型把整串视为不可拆解的术语。
- **程序员/编程符号**（03）："逗号"仍然替代"分号"——补的样本不够多。
- **网络梗的语义层**（07）：奶茶续命这条，三轮模型都没真正理解"躺平/起飞"是反讽。

#### 🆕 Run #3 新出现的小问题

- 02 短视频："私信"（莫名其妙的替换，Run #1 的"短影音"是高峰，#2 退化为"抖音"，#3 又跑偏到"私信"）
- 03 "屡试不爽"误用（成语本意是"屡次试验都不错"，但被模型当成"反复失败"）

### 结论

- **Run #3 是综合最佳产物**，作为项目默认 adapter 发布。
- 折中超参（rank=6, dropout=0.02）的确在表达力与稳定性之间找到了好平衡。
- **盲区根因揭示**：剩下的三大盲区（百分数、编程符号、网络梗反讽）不是超参问题，是**数据层面的 prompt 表面形态匹配不上**。下次补数据应该针对完全一致的表面形态。
